## 2. Структура проекта

Проект организован в следующей структуре:

- `pytest.ini` – конфигурация pytest.
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.
- `requirements/base.txt` – зависимости проекта (библиотеки Python, необходимые для запуска) для прод среды.
- `requirements/dev.txt` – зависимости проекта (библиотеки Python, необходимые для запуска) для дев среды.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.13`.
- Установленные системные зависимости (`docker`).

### 3.2. Установка окружения

Сценарий `Windows`:

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv
.venv\Scripts\activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements/dev.txt
```

Сценарий `macOS`/`Linux`:
```bash
# Перейти в папку проекта
cd project

python3 -m venv .venv
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements/dev.txt
```

## 4. Как запустить проект

Для запуска экспериментов нужно скачать датасет из пункта 5.

Проект можно запустить двумя способами:

* локально (Python + venv)
* через Docker (рекомендуется, так как требуется PostgreSQL и S3-совместимое хранилище MinIO)

---

## 4.1. Локальный запуск (API)

### 1. Подготовка окружения

Скопируйте файл окружения и заполните переменные:

```bash
cp ./configs/.env.example ./configs/.env
```

Обязательно укажите:

* `SECRET_KEY` (любая строка из английских символов и цифр)

---

### 2. Запуск инфраструктуры (PostgreSQL + MinIO)

```bash
docker compose -p ai-project -f configs/docker-compose.yaml up -d postgres s3 create-bucket
```

---

### 3. Запуск приложения

```bash
cd project
source .venv/bin/activate  # при необходимости
python -m src
```

API будет доступен на:

```
http://localhost:8000
```

---

## 4.2. Запуск через Docker (рекомендуется)

### 1. Подготовка окружения

```bash
cp ./configs/.env.example ./configs/.env.dev.example
```

Важно:

* замените `localhost` в настройках на имена сервисов Docker (`postgres`, `s3`, и т.д.)
* укажите `SECRET_KEY` (любая строка из английских символов и цифр)

---

### 2. Запуск всего проекта

```bash
docker compose -p ai-project -f configs/docker-compose.yaml up -d
```

## 4.4. API эндпоинты

### Analytic

| Метод  | Endpoint                        | Описание                     |
| ------ | ------------------------------- | ---------------------------- |
| POST   | `/api/v1/analytic/`             | загрузка датасета (CSV/XLSX) |
| POST   | `/api/v1/analytic/{id}/analyze` | запуск анализа               |
| GET    | `/api/v1/analytic/`             | список аналитик              |
| GET    | `/api/v1/analytic/{id}`         | получение аналитики          |
| GET    | `/api/v1/analytic/{id}/full`    | полная аналитика             |
| DELETE | `/api/v1/analytic/{id}`         | удаление аналитики           |

---

### Predict

| Метод  | Endpoint                      | Описание                         |
| ------ | ----------------------------- | -------------------------------- |
| POST   | `/api/v1/predict/`            | загрузка файла для предсказания  |
| GET    | `/api/v1/predict/{id}`        | статус предсказания              |
| GET    | `/api/v1/predict/`            | список предсказаний (pagination) |
| GET    | `/api/v1/predict/{id}/report` | скачать отчет                    |
| DELETE | `/api/v1/predict/{id}`        | удалить предсказание             |
